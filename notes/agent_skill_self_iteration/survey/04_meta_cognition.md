# Meta-Cognition & Reflection

> 调研日期：2026-06-18

## 核心发现

- **Self-Refine 范式已成基础**：同一 LLM 扮演 Generator/Feedback/Refiner 三角色，7 项任务平均 ~20% 提升。但会放大 **self-bias** (2402.11436)，更大模型和外部反馈可缓解
- **元认知架构呈分层趋势**：
  - **Sophia** 的 System 3 — 持久身份层，高于 System 1（直觉）和 System 2（推理）
  - **SMARt** — 四层状态机，显式管理推理状态转换
  - **DS-MCM** — 双速监控机制，快速模式（Low-Level Monitor）+ 慢速模式（High-Level Monitor）
- **元认知与 RL 深度耦合**：E-SPL (2602.14697) 联合进化系统提示 + RL 权重；MMPO (2605.30159) 信念熵自监督
- **CoT 从链式 → 树 → 图**：Tree of Thoughts (2305.10601) 树搜索多路径推理；ReasoningFlow (2606.05402) DAG 推理轨迹
- **7 种通用推理算子 (ReasonOps, 2605.29192)**：backtracking, inferring, hypothesizing, deducing, analogizing, decomposing, verifying
- **LLM 规划是短视的**：Extracting Search Trees (2605.06840) 性能由搜索广度而非深度预测
- **小模型元认知局限**："Right for Wrong Reasons" (2601.00513) 发现 50-69% 正确回答含缺陷推理，self-critique 在弱模型上反而有害
- **自验证从被动 → 主动**：SmartSnap (2512.22322) 主动现场验证；FineVerify (2606.00660) 细粒度子问题分解验证
- **多 Agent 反思缓解退化**：MAR (2512.20845) 单 LLM 连续反思陷入思维退化，多 Agent 辩论产生多样化反思
- **置信度校准与弃权**：Pause and Reflect (2605.14098) 共形风险控制，5% 弃权率下选择性准确率 90.1%

## 自验证机制

| 机制 | 代表工作 | 核心思路 |
|------|----------|---------|
| **被动自验证** | Self-Refine (2303.17651) | 同一 LLM 生成 → 自我反馈 → 精炼，有 self-bias 风险 |
| **主动现场验证** | SmartSnap (2512.22322) | 在推理过程中主动插入验证点，收集环境证据 |
| **细粒度子问题分解** | FineVerify (2606.00660) | 将复杂验证拆解为可验证的子问题，逐层搜索 |
| **外部验证器** | V-STaR (DPO), ARIS (adversarial review) | 引入独立验证模型或对抗审查，避免递归漂移 |
| **自验证代码 Agent** | ReVeal (2506.11442) | 代码 Agent 在自我改进循环中验证行为，支持 20+ 轮进化 |
| **共形风险控制** | Pause and Reflect (2605.14098) | 统计保证的弃权决策，校准置信度 |

### 关键教训
- **外部反馈优于自我反馈**：自我反馈单独导致递归漂移；外部验证器/批评者必不可少（跨论文共识）
- **自验证引用易被操纵**：80% 自验证引用可被操纵值欺骗 (2603.12564)
- **验证器组件是自迭代系统的必备组件** — 所有成功的自迭代系统均包含验证器

## 推理轨迹分析

### 推理结构演化
| 阶段 | 代表 | 结构 | 特点 |
|------|------|------|------|
| 链式 | Chain-of-Thought | 线性 chain | 单路径推理 |
| 树式 | Tree of Thoughts (2305.10601, NeurIPS 2023) | 树搜索 | 多路径探索+回溯 |
| 图式 | ReasoningFlow (2606.05402) | DAG | 子任务独立执行+条件聚合 |

### ReasonOps — 7 种通用推理算子 (2605.29192)
1. **Backtracking** — 回溯，撤销无效路径
2. **Inferring** — 推断，从已知推导未知
3. **Hypothesizing** — 假设，提出可验证的猜想
4. **Deducing** — 演绎，逻辑必然推导
5. **Analogizing** — 类比，跨域知识迁移
6. **Decomposing** — 分解，拆分为子问题
7. **Verifying** — 验证，确认推理正确性

### 轨迹蒸馏
- **Socratic-SWE** (2606.07412)：从历史求解轨迹提取结构化 skill，指导生成针对性修复任务，SWE-bench Verified 50.40% (3 轮)
- **CODESKILL** (2605.25430)：RL 管理 skill 提取 + skill-bank 维护，密集+稀疏混合奖励
- **MemCoder** (2603.13258)：意图-代码映射 + 自我修正 + 经验内化，结构化解码经验

## 元认知架构模式

### Sophia — System 3 元认知层 (2512.18202)
- **三层架构**：System 1（直觉/快速响应）→ System 2（推理/规划）→ System 3（元认知/持久身份层）
- **System 3 功能**：持久身份层，管理 agent 的认知历史、价值观、长期目标
- **核心创新**：将元认知提升到与直觉、推理并列的架构层次

### SMARt — 四层状态机
- **路由层**：任务分发（Routing）
- **规划层**：策略生成（Planning）
- **执行层**：动作执行（Execution）
- **监控层**：结果评估（Monitoring）
- **特点**：显式状态转换 + 错误恢复机制

### DS-MCM — 双速监控
- **Low-Level Monitor**：快速模式，实时动作级监控，低开销
- **High-Level Monitor**：慢速模式，全局策略级评估，高开销
- **协同**：快慢结合，平衡实时性与深度分析

### Agent 自迭代架构模式 (6 种)
| 模式 | 适用场景 | 代表工作 |
|------|---------|---------|
| 口头 RL（语言反馈+记忆） | 单步决策改进 | Reflexion |
| 轨迹蒸馏 | 多步工作流优化 | Socratic-SWE, CODESKILL |
| 三阶段自验证 | 输出质量保证 | Self-Refine, FineVerify |
| 群组进化 | 多 Agent 协作优化 | GEA |
| 递归自我设计 | 架构层进化 | MetaAI Recursive |
| 无标注自优化 | 无验证集场景 | RHO |

## 安全与偏见风险

### Self-Bias (自我偏好偏差)
- **发现**：Self-Refine 会放大 self-bias (2402.11436)，模型倾向于接受自己的初始输出，即使自我反馈指出问题
- **缓解**：更大模型 + 外部权威反馈可缓解；多 Agent 辩论（MAR）可产生多样化反思
- **机制**：递归放大初始偏差，形成**思维退化 (Degeneration of Thought)**

### 思维退化
- **单 LLM 连续反思** → 陷入思维退化 (MAR, 2512.20845)，反思质量逐轮下降
- **多 Agent 辩论** → 产生多样化反思，打破同质化思维模式
- **MetaAI Recursive** (2606.09663)：80 次递归迭代中可能产生对抗行为或能力退化

### 探索-利用退化
- 探索能力在迭代中迅速退化（B-STaR, HS-STaR, AdaSTaR）
- 需动态分配采样预算到边界难度，防止 agent 停留在局部最优

### 安全退化
- ANCHOR 发现自主进化可能导致能力退化和安全漂移
- Zombie Agents (2602.15654)：长期记忆和 session 间记忆需安全隔离

### 技能组合风险
- 22.25% 技能对存在组合风险 (2606.00448)，单个无害 skill 组合后可能有害
- 26.1% 社区 skill 存在漏洞 (arXiv:2601.10338)

## 与 optics_agent 的关系

### 当前状态
- **元认知/反思**：仅停留在 `final_report.md` 事后文档，属于**中等差距**
- **自验证**：人工判断 success 类型（matrix_factorization_failed, surrogate_fallback 等）
- **推理轨迹**：无轨迹记录和分析

### 直接可采纳的改进
1. **反射式失败编码**（4 小时）— 将 Degiron v2 矩阵分解失败模式正式写入 skill，形成结构化的反思触发机制
2. **运行中反思**（中期）— 引入 Reflexion 风格三元组（Actor/Evaluator/Memory），每次 COMSOL 任务执行后自动反思
3. **自验证节点**（中期）— 程序化检查 `Im(neff)` 非零、数值范围、`surrogate_fallback` 标记
4. **多 Agent 辩论**（长期）— paper-reader / model-builder / result-validator 多角色辩论验证物理结果

### 核心设计原则
- **外部验证器不可或缺**：科学计算领域尤其需要物理一致性验证器，而非仅依赖自我反馈
- **物理验证是终极瓶颈**：自动化架构再先进，没有 GR 团队的标准答案也无法判断进展
- **轨迹蒸馏远期的蓝图方向**：每次成功的 COMSOL 仿真自动生成 skill，从失败轨迹提取修复策略
