## 核心发现

科学计算 Agent 正在形成独立于通用 SWE Agent 的子方向。Skills-first 架构特别适合需要多步工作流的科学计算场景。关键区别在于：科学计算 Agent 的目标是**物理正确**而非功能正确，验证方式依赖物理规律一致性而非单元测试，计算成本高（小时级仿真 vs 秒级），且必须确定性复现。

### 关键趋势

1. **科学/工程领域的 Agent 技能系统正在形成独立子方向**（Section 2.5）
2. **技能依赖图与多物理场工作流天然匹配** — SkillGraph vs COMSOL 多物理场耦合
3. **形式化验证对科学计算可靠性至关重要** — SEVerA 相关工作
4. **外部验证器不可或缺** — 科学计算结果必须经物理规律检验
5. **代码作为统一技能表示** — COMSOL Java 是 optics_agent 的天然技能载体

## 科学计算 Agent 的特殊需求

### 与通用 SWE Agent 的关键区别（Section 6.2）

| 维度 | SWE Agent | 科学计算 Agent (optics_agent) |
|------|-----------|------------------------------|
| **目标** | 功能正确 | 物理正确 |
| **验证方式** | 单元测试/集成测试 | 物理规律一致性 + 数值收敛性 |
| **工作流特点** | 逐步修复 | 参数化流水线 |
| **计算成本** | 低（秒级） | 高（小时级仿真） |
| **领域壁垒** | 编程语言语法 | 多物理场耦合 + API 深度 |
| **可复现性要求** | 可接受概率性 | 必须确定性能复现 |
| **从失败学习** | bug 修复模式 | 仿真配置参数调优 |

### 特殊设计需求总结（Section 6）

1. **物理验证是终极瓶颈** — 自动化架构再先进，没有 GR 团队的标准答案也无法判断进展
2. **外部验证器不可或缺** — 自反馈导致递归漂移（Self-Bias 2402.11436），科学计算必须外部验证
3. **形式化验证保障** — SEVerA 的 GRPO+验证范式对科学计算可靠性至关重要（Section 2.8）
4. **从失败中学习模式不同** — SWE Agent 修复 bug，科学计算 Agent 调优仿真配置参数
5. **领域知识编码是最大壁垒** — SADE 提出从领域知识到技能库的系统方法论

## 对 optics_agent 的启示

### Mapped Insights（Section 5 & 6）

| 前沿工作 | 核心思想 | optics_agent 对应 | 差距 | 优先级 |
|----------|---------|-------------------|------|--------|
| **Voyager** | 自动技能发现/迭代 | 现有 8 个手工 skill | 无自动技能创建 | 中长期 |
| **Reflexion** | 自我反思+重试 | `final_report.md` 事后文档 | 无运行中反思 | 中期 |
| **Socratic-SWE** | 轨迹蒸馏→自演化 | 无对应 | 缺失闭环引擎 | 长期 |
| **EvoDS** | 科学工作流自动化 | 单入口 submit_comsol.py | 无自动化管道 | 中期 |
| **SkillGraph** | 技能依赖图 | 无依赖建模 | 多物理场耦合依赖未管理 | 中期 |
| **NORA** | Skills-first 科学架构 | AGENTS.md 路由表 | 无动态注册/发现 | 中期 |
| **SADE** | 领域知识编码 | 笔记形式知识 | 非结构化，不可程序化调用 | 中期 |
| **SEVerA** | 形式化验证 | 人工判断 success 类型 | 无自动验证 | 长期 |

### 深层启示（Section 6.3）

1. **COMSOL Java 是 optics_agent 的天然技能载体** — Code-as-Skill 范式的最佳映射，可直接采用 Voyager 模式
2. **Voyager 的"自动课程"可映射为"自动复杂度递增"** — 简单几何 → 多物理场耦合 → 全参数扫描
3. **Socratic-SWE 的轨迹蒸馏闭环** → 每次 COMSOL 仿真自动优化下一个 skill
4. **DSPy 编译器式 LM 编程哲学** → optics_agent 用声明式 YAML 替代隐式 prompt 编排
5. **物理验证是终极瓶颈** — 自动化架构再先进，没有 GR 团队的标准答案也无法判断进展
6. **外部验证器不可或缺** — Reflexion/Self-Refine/SWE-agent 共识：自反馈导致递归漂移，需外部批评者

### 关键开放性问题的引用（Section 6.4）

- "自我反馈单独使用会导致递归漂移"（Self-Bias, 2402.11436）
- "Agent 框架设计比底层模型更重要"（Claw-SWE, 2606.12344，性能差 6x）
- "探索能力在迭代中迅速退化"（B-STaR, AdaSTaR）
- "元 Agent 能力仍有限，很少匹敌人类专家"（MAC, 2606.04455）
- "80% 自验证引用可被操纵值欺骗"（2603.12564）
